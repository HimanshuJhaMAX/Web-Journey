Essential Tags (Keywords)

<html> Root Element
Syntax
<html>
  ...content...
</html>
Example
<html lang="en">
</html>
Use Case: Serves as the container for all other HTML elements on the page (except the <!DOCTYPE>). It is the root of the document.
<head> Metadata Container
Syntax
<head>
  ...meta tags, title, links...
</head>
Example
<head>
  <title>My Page</title>
</head>
Use Case: Contains machine-readable information (metadata) about the document, like its title, scripts, and stylesheets. This content is NOT visible on the page itself.
<body> Document Body
Syntax
<body>
  ...visible content...
</body>
Example
<body>
  <h1>Hello World!</h1>
</body>
Use Case: Encloses all the content that you actually want to display in the web browser window (text, images, links, etc.).
<h1> - <h6> Headings
Syntax
<h1>Heading Text</h1>
Example
<h1>Main Title</h1>
<h2>Subtitle</h2>
Use Case: Used to define headings for sections of content. <h1> is the most important (main title), down to <h6> (least important). Great for SEO and structure.
<p> Paragraph
Syntax
<p>Text content goes here.</p>
Example
<p>Welcome to my website.</p>
Use Case: Defines a block of text. It automatically adds some space (margin) before and after the text.
<a> Anchor (Link)
Syntax
<a href="URL">Link Text</a>
Example
<a href="https://google.com">Google</a>
Use Case: Creates a hyperlink to another web page, file, email address, or location in the same page.
<img> Image
Syntax
<img src="image.jpg" alt="Description">
Example
<img src="logo.png" alt="Company Logo">
Use Case: Embeds an image into the webpage. It is a self-closing tag (does not need a closing tag).
<div> Division
Syntax
<div>...content...</div>
Example
<div class="container">
  <p>Grouped text</p>
</div>
Use Case: A generic block-level container used to group other elements together, usually for styling purposes with CSS or targeting with JavaScript.
<span> Inline Container
Syntax
<span>...content...</span>
Example
<p>I am <span style="color:red">red</span>.</p>
Use Case: A generic inline container used to style a small portion of text within a larger block, without breaking onto a new line.
<ul> & <li> Unordered List
Syntax
<ul>
  <li>Item</li>
</ul>
Example
<ul>
  <li>Apple</li>
  <li>Banana</li>
</ul>
Use Case: Used to create bulleted lists where the order of items does not matter.

Form Tags

<form> Form Container
Syntax
<form action="/submit" method="POST">
  ...inputs...
</form>
Example
<form>
  <input type="text">
</form>
Use Case: Wraps input elements to submit user data to a server.
<input> Input Field
Syntax
<input type="type" name="name">
Example
<input type="email" placeholder="Email">
<input type="checkbox">
Use Case: A versatile field for user input (text, checkbox, radio, password, etc.).
<button> Button
Syntax
<button type="button">Text</button>
Example
<button type="submit">Send</button>
Use Case: A clickable button used to submit forms or trigger JavaScript actions.
<label> Input Label
Syntax
<label for="id">Text</label>
Example
<label for="user">Username</label>
<input id="user">
Use Case: Provides a label for an input. Clicking the label focuses the input. Crucial for accessibility.
<textarea> Multi-line Input
Syntax
<textarea rows="4"></textarea>
Example
<textarea placeholder="Message"></textarea>
Use Case: Used when you need the user to input multiple lines of text, like a comment or message.
<select> & <option> Dropdown List
Syntax
<select>
  <option value="v">Text</option>
</select>
Example
<select>
  <option>Red</option>
  <option>Blue</option>
</select>
Use Case: Creates a dropdown menu for users to select one (or more) options.

Table Tags

<table> Data Table
Syntax
<table>...rows...</table>
Example
<table border="1">
  <tr><td>Data</td></tr>
</table>
Use Case: Defines a table to display tabular data (rows and columns). Do not use for page layout.
<tr>, <th>, <td> Row, Header, Data
Syntax
<tr>
  <th>Header</th>
  <td>Cell</td>
</tr>
Example
<tr>
  <td>Row 1, Cell 1</td>
</tr>
Use Case: `tr` creates a row. `th` creates a bold/centered header cell. `td` creates a standard data cell.
<thead>, <tbody> Table Sections
Syntax
<thead>...</thead>
<tbody>...</tbody>
Example
<thead><tr><th>Title</th></tr></thead>
Use Case: Groups table content. `thead` for headers, `tbody` for main content, `tfoot` for summaries.

Semantic & Layout Tags

<header> & <footer> Page Sections
Syntax
<header>...</header>
<footer>...</footer>
Example
<header><h1>Logo</h1></header>
Use Case: `header` for introductory content/nav. `footer` for copyrights, links at the bottom.
<main> Main Content
Syntax
<main>...content...</main>
Example
<main>
  <article>Blog Post</article>
</main>
Use Case: Specifies the dominant content of the `body`. Should be unique per page.
<section> & <article> Content Blocks
Syntax
<section>...</section>
<article>...</article>
Example
<article>
  <h2>News</h2>
</article>
Use Case: `section` groups related thematic content. `article` is for independent, self-contained content (like a blog post).
<nav> & <aside> Navigation & Sidebar
Syntax
<nav>...</nav>
<aside>...</aside>
Example
<nav><a href="/">Home</a></nav>
Use Case: `nav` for main navigation links. `aside` for content indirectly related to the main content (sidebars).

Media Tags

<audio> & <video> Media Players
Syntax
<audio controls src="a.mp3"></audio>
<video src="v.mp4"></video>
Example
<video controls width="500">
  <source src="movie.mp4">
</video>
Use Case: Embeds sound or video content directly into the webpage with native browser controls.
<iframe> Inline Frame
Syntax
<iframe src="url"></iframe>
Example
<iframe src="https://youtube.com/..." width="560" height="315"></iframe>
Use Case: Embeds another HTML document (like a YouTube video, Google Map, or external site) within the current page.

Common Attributes

id Unique Identifier
Syntax
<tag id="unique-name">
Example
<h1 id="main-heading">Welcome</h1>
Use Case: Provides a unique name for an element. Used to style that specific element with CSS or manipulate it with JavaScript. Must be unique per page.
class Class Identifier
Syntax
<tag class="class-name">
Example
<p class="highlight text-bold">Text</p>
Use Case: Assigns one or more class names to an element. Used by CSS and JS to target multiple elements that share the same styling or behavior.
src Source
Syntax
<tag src="path/to/file">
Example
<img src="cat.jpg">
<script src="app.js"></script>
Use Case: Specifies the path/URL to an external resource, typically used with <img>, <script>, <iframe>, and <audio>/<video>.
href Hypertext Reference
Syntax
<a href="url">
Example
<a href="/about.html">About Us</a>
<link href="style.css" rel="stylesheet">
Use Case: Specifies the URL of the page the link goes to (used on <a>) or the location of an external resource (used on <link>).
alt Alternative Text
Syntax
<img alt="description">
Example
<img src="dog.jpg" alt="A brown dog running">
Use Case: Provides fallback text if an image fails to load. Crucial for accessibility (screen readers) and SEO.
type Input Type
Syntax
<input type="type">
Example
<input type="password">
<button type="submit">
Use Case: Defines the specific kind of input field (e.g., text, email, number) or button behavior.
value & placeholder Input Data
Syntax
<input value="val" placeholder="hint">
Example
<input placeholder="Search...">
Use Case: `value` sets the default/current text of an input. `placeholder` shows a temporary hint before typing starts.
required & disabled Boolean States
Syntax
<input required disabled>
Example
<button disabled>Processing</button>
Use Case: `required` forces the user to fill a field before submitting. `disabled` prevents interaction.
style Inline CSS
Syntax
<tag style="property: value;">
Example
<p style="color: red; margin-top: 10px;">
Use Case: Applies CSS directly to a single element. (Use sparingly; prefer external classes).
target Link Target
Syntax
<a target="_blank">
Example
<a href="..." target="_blank">Open in new tab</a>
Use Case: Tells the browser where to open the linked document. `_blank` opens it in a new tab/window.

Common Entities

Entities are used to display reserved characters (like < and >) or invisible characters.

Entity Name Syntax Result / Example Use Case
&lt; &lt; < (Less than) Used to display a < symbol without the browser thinking it's the start of an HTML tag.
&gt; &gt; > (Greater than) Used to display a > symbol without the browser thinking it's the end of an HTML tag.
&amp; &amp; & (Ampersand) Used to safely display an ampersand in text or URLs.
&copy; &copy; © (Copyright) Displays the copyright symbol, usually found in the footer of a webpage.
&nbsp; &nbsp; (Non-breaking space) Creates a space that prevents a line break at its position. Useful for spacing or keeping two words together.
&euro; / &pound; / &yen; &euro; € / £ / ¥ Currency symbols for Euro, Pound, and Yen.
&rarr; / &larr; &rarr; → (Right Arrow) / ← Directional arrows useful for navigation links or indicators.
&trade; / &reg; &trade; / &reg; ™ (Trademark) / ® Legal and brand symbols for trademark and registered trademark.
&quot; / &apos; &quot; / &apos; " (Quote) / ' (Apostrophe) Double and single quotation marks (useful if you're inside an attribute value).

Common Required Words / Concepts

DOM (Document Object Model)

An programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages (like JavaScript) can interact with the page.

Element

An individual component of an HTML document. An element usually consists of a start tag, content, and an end tag (e.g., <p>This is an element</p>). Sometimes elements are empty and self-closing (e.g., <img>).

Tag

The markup that denotes the start and end of an element. Tags are enclosed in angle brackets. For example, <h1> is the opening tag and </h1> is the closing tag.

Semantic HTML

Using HTML markup to reinforce the semantics, or meaning, of the information in webpages rather than merely to define its presentation (look). Examples include using <article>, <nav>, and <footer> instead of generic <div> tags. It aids accessibility and SEO.

Block-level vs. Inline Elements

Block-level elements (like <div>, <p>, <h1>) always start on a new line and take up the full width available. Inline elements (like <span>, <a>, <strong>) do not start on a new line and only take up as much width as necessary.
